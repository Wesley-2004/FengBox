## 写在前面

下面这些 **prompt** 不是给 FengBox 工具用的，而是给你日常用 ChatGPT / Claude / DeepSeek 等通用 AI 时参考的模板。你可以根据自己场景修改关键词。

---

## 📝 文案写作类

### 公众号文章开头

```
扮演一位资深公众号编辑。
为【主题：xxx】写 3 种不同风格的公众号开头（200 字内）：
1. 故事钩子
2. 反常识观点
3. 数据震撼
要求：每种风格都给出可以直接用的开头文字，不要解释。
```

### 朋友圈文案

```
为"【主题：xxx】"写 5 条朋友圈文案，每条 30-50 字：
- 第 1 条：让人忍不住点赞
- 第 2 条：让人想评论
- 第 3 条：转发出乎意料
- 第 4 条：克制有深意
- 第 5 条：纯分享不加修饰
风格：朋友间真实自然，不要营销腔。
```

## 💼 工作效率

### 会议纪要整理

```
我给你一段会议录音的文字稿。请输出：
1. 3 行 summary（最重要的 3 件事）
2. 决议事项列表（清晰的 ToDo）
3. 待办分配表（人 + 任务 + 截止日）

[粘贴会议文字稿]
```

### 邮件润色

```
把下面这段话改写成更专业、清晰、礼貌的工作邮件。要求：
- 不改变原意
- 段落清晰
- 称呼/落款要正式

[粘贴原文]
```

## 🎓 学习类

### 概念讲解

```
用 3 个层级向不同人解释【xxx】这个概念：
1. 给完全外行的 8 岁小孩
2. 给对这个领域有点兴趣的大学生
3. 给这个领域的从业者（要包含专业术语）
每个层级 100 字以内。
```

### 读书笔记整理

```
以下是我对【书名/章节】的笔记片段，请帮我整理成结构化笔记：
- 核心观点（3 条）
- 金句（5 条）
- 我自己的疑问（AI 可以补充回答）
- 实践建议（如何应用到我的工作/生活）

[粘贴笔记]
```

## 💻 编程类

### 代码 review

```
你是高级工程师，帮我 review 下面这段代码，重点关注：
1. 潜在的 bug
2. 性能问题
3. 可读性
4. 边界情况
给具体行号 + 改写建议。

[粘贴代码]
```

### 报错解释

```
我运行命令：[粘贴命令]
报错：[粘贴错误信息]
我的环境：[操作系统/语言版本]
请先解释这个错误最可能的原因（按概率排序），然后给出 3 个排查方向。
```

## 🍳 日常生活

### 旅行规划

```
我打算【目的地】旅行【天数】天，【月份】出行，预算【xxx】元。
兴趣：列出 3-5 个
请给我每天的行程大纲（不要具体时刻，但每天要有 1 个亮点活动 + 1 个美食推荐）。
```

### 决策辅助

```
我要在【选项 A】和【选项 B】之间做决定。
我的偏好：[列出 3-5 个你最看重的因素]
请你用决策矩阵的形式列出每个选项在每个因素上的优劣（1-5 分），最后给一个建议。
但最终决策权在你自己。

选项 A：[描述]
选项 B：[描述]
```

## ⚠️ 使用提示

1. **越具体越好** — 给 AI 详细的背景（你是谁、给谁看、要发布到哪），效果会大幅提升
2. **分段给指令** — 复杂任务拆成多轮，比一次性长 prompt 效果好
3. **让 AI 解释** — 不确定的地方问"为什么这样写"，帮你理解而不是盲从
